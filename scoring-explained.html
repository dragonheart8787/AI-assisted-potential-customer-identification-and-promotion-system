評分機制說明

對應程式：website-analyzer.js、ai-assistant.js（classifyCustomer）。分數皆為 0–100；下列**總表**為判斷依據，下方為加權細節。

總表：分數 × 判斷依據（給審閱者對照，非僅命名）

分數判斷依據
website
need_score		 CTA（聯絡／預約／詢價等＋連結或按鈕）、聯絡表單、SEO title／description（含 H1）、網站速度（首頁 >3 秒加分）、手機版適配（viewport）；另：版權年份過舊、作品／服務頁線索。
security
need_score		 HTTPS、security.txt、基本安全標頭（HSTS、X-Content-Type-Options、X-Frame-Options、CSP）、robots／sitemap／隱私聲明（HTML 線索）、外露技術資訊、WordPress 無 CSP 等。
lead_score 決策者線索（職稱）、需求標籤（CRM tags）、互動紀錄（次數）、網站分析訊號（若有）；聯絡完整度可擴充，目前規則以職稱／標籤／互動／網站訊號為主。

一、website_need_score（網站撰寫／改版需求）— 加權細節

依首頁 HTML 與載入時間加權累加，上限 100。

條件加分偵測方式
缺少 CTA（聯絡、預約、詢價等關鍵字 + 連結/按鈕）+20正則 + DOM 線索
缺少表單+15<form> + submit
缺少 viewport（行動版）+15meta viewport
版權年份過舊（>2 年）+10footer 年份
缺少 meta title+10SEO
缺少 meta description+10SEO
缺少 H1+5SEO 結構
缺少作品/服務/關於等頁面線索+10連結文字關鍵字
載入 > 3 秒+10計時

二、security_need_score（資安強化需求）— 加權細節

越高代表越需要資安健檢或設定補強。

條件加分說明
未使用 HTTPS+30URL 協定
缺少 HSTS+15Response header
缺少 X-Content-Type-Options+10標頭
缺少 X-Frame-Options+10標頭
無 security.txt+10/.well-known/security.txt
外露技術資訊+10Server、WP 版本字串等
WordPress 且無 CSP+15技術堆疊 + 標頭

HTML 內亦會檢查 robots、sitemap、privacy 等文字線索（見 analyzeSecurity），主要加權仍以上表為主。

三、lead_score（客戶優先級／AI 分類）— 加權細節

基準分 50，再依職稱、網站分析、互動、標籤調整，最後 clamp 至 0–100。

面向規則摘要典型加分/扣分
職稱／決策者CEO、Founder、Chief → 高階；VP、Director → 中高階+25 / +15
網站分析issues ≥3；security 訊號；seo 訊號+15 / +10 / +5
互動次數>5 次高互動；>2 次略加分+15 / +5
標籤高價值、熱門潛在客戶；有興趣；拒絕、冷名單+20、+10 / -30
聯絡完整度（概念）實作上可搭配 CRM：email、電話、官網齊全者後續可加權；目前以標籤與互動間接反映—

分類結果：high_value（≥80）、interested（≥60）、standard（≥40）、cold（<40）。並產出 need_type、urgency、outreach_angle 供推廣話術使用。

四、與 leadSignals 的關係

website_need_score、security_need_score 計算後會寫入分析結果，並驅動 extractLeadSignals（例如網站需求 ≥50、資安需求 ≥50 時產生對應訊號），供 CRM 與報告引用。
