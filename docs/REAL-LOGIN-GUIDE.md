# 🚀 真實社群媒體登入使用指南

## 📋 概述

本系統現在支援真實的社群媒體API登入，可以連接到Twitter/X、LinkedIn、Instagram和Facebook的真實API，實現真正的訊息發送功能。

## 🔧 設定步驟

### 第一步：獲取API金鑰

#### Twitter/X API
1. 前往 [Twitter開發者平台](https://developer.twitter.com/)
2. 創建一個新的應用程式
3. 獲取以下資訊：
   - API Key
   - API Secret
   - Bearer Token

#### LinkedIn API
1. 前往 [LinkedIn開發者平台](https://developer.linkedin.com/)
2. 創建一個新的應用程式
3. 獲取以下資訊：
   - Client ID
   - Client Secret

#### Instagram API
1. 前往 [Facebook開發者平台](https://developers.facebook.com/)
2. 創建一個新的應用程式
3. 添加Instagram Basic Display產品
4. 獲取以下資訊：
   - App ID
   - App Secret

#### Facebook API
1. 前往 [Facebook開發者平台](https://developers.facebook.com/)
2. 創建一個新的應用程式
3. 獲取以下資訊：
   - App ID
   - App Secret

### 第二步：配置API設定

1. 在主頁面點擊「🔐 API設定」按鈕
2. 在開啟的設定頁面中填入相應的API金鑰
3. 點擊「💾 保存所有設定」
4. 使用「🧪 測試所有API」功能驗證配置

### 第三步：切換到真實登入模式

1. 在主頁面點擊「🚀 切換真實登入」按鈕
2. 確認切換到真實登入模式
3. 系統會自動更新登入按鈕功能

## 🔐 真實登入流程

### Twitter/X 登入
- 使用Bearer Token進行API認證
- 直接驗證API連接
- 無需OAuth流程

### LinkedIn 登入
- 啟動OAuth 2.0認證流程
- 在新視窗中完成授權
- 自動獲取訪問令牌

### Instagram 登入
- 啟動OAuth 2.0認證流程
- 在新視窗中完成授權
- 自動獲取訪問令牌

### Facebook 登入
- 啟動OAuth 2.0認證流程
- 在新視窗中完成授權
- 自動獲取訪問令牌

## 📤 真實訊息發送

### 發送流程
1. 選擇目標領袖
2. 編寫訊息內容
3. 點擊發送按鈕
4. 系統會檢測已認證的帳號
5. 選擇使用真實帳號或模擬發送
6. 系統會通過真實API發送訊息

### 支援的訊息類型
- **Twitter**: 直接訊息 (DM)
- **LinkedIn**: InMail訊息
- **Instagram**: 直接訊息
- **Facebook**: 直接訊息

## 🛠️ 故障排除

### 常見問題

#### Q: API金鑰配置後仍無法登入
A: 
1. 檢查API金鑰是否正確
2. 確認應用程式狀態是否正常
3. 檢查網路連接
4. 查看瀏覽器控制台錯誤訊息

#### Q: OAuth流程被中斷
A:
1. 確保彈出視窗沒有被阻擋
2. 檢查瀏覽器設定
3. 重新嘗試登入流程

#### Q: 訊息發送失敗
A:
1. 檢查帳號是否已認證
2. 確認目標用戶ID是否正確
3. 檢查API使用限制
4. 查看錯誤訊息詳情

### 錯誤代碼說明

| 錯誤代碼 | 說明 | 解決方案 |
|---------|------|----------|
| 401 | 未授權 | 檢查API金鑰是否正確 |
| 403 | 禁止訪問 | 檢查應用程式權限設定 |
| 429 | 請求過多 | 等待一段時間後重試 |
| 500 | 伺服器錯誤 | 稍後重試或聯繫平台支援 |
