# 推廣中心 - 成交 SOP 營運 Playbook

> 不整理就會每次都重打一次仗。  
> 本 Playbook 對應系統既有功能，可直接執行。

---

## 一、名單 SOP

### 1.1 每天抓幾筆

| 階段 | 每日新增 | 說明 |
|------|----------|------|
| 冷啟動 | 10–20 筆 | 先驗證話術與轉換率 |
| 穩定期 | 30–50 筆 | 依回覆率調整 |
| 擴張期 | 50–100 筆 | 需有跟進與會議 SOP 支撐 |

**原則**：寧可少而精，不要多而亂。每天能認真跟進的數量才是有效名單。

### 1.2 來源優先順序

| 優先級 | 來源 | 理由 | 系統對應 |
|--------|------|------|----------|
| P1 | 官網聯絡頁抓取 | 已有官網、有聯絡意圖 | `website-analyzer.extractContactInfo` |
| P1 | 招募頁（104/1111） | 招聘 = 有預算、有需求 | 發現客戶 → 招募頁 |
| P2 | Google Places | 本地商家、有地址電話 | 發現客戶 → Google Maps |
| P2 | Facebook Places | 社群活躍商家 | 發現客戶 → 社群媒體 |
| P3 | 手動新增 / CSV | 展覽、協會、轉介紹 | 發現客戶 → 手動 / CSV |

**每日建議配比**：P1 佔 60%、P2 佔 30%、P3 佔 10%。

### 1.3 分數多少以上才打

| 分數 | 行動 | 說明 |
|------|------|------|
| website_need_score ≥ 60 | 優先打 | 網站問題多，需求明確 |
| security_need_score ≥ 60 | 優先打 | 資安需求高，可推資安健檢 |
| lead_score ≥ 70 | 優先打 | AI 分類高分，決策者或高互動 |
| 50–69 | 可打 | 依產業與來源決定 |
| &lt; 50 | 暫緩 | 除非有強需求訊號（如招募資安職缺） |

**實務**：先打 website_need_score + security_need_score 雙高，再打單一高分，最後打 50–69。

## 二、聯繫 SOP

### 2.1 第一天發什麼

**主旨**：簡短、個人化、有鉤子  
範例：`{公司名} 官網的 3 個小問題 - 可免費幫你看`

**內容結構**（約 150 字）：
1. 開頭：一句話說明為何找他（產業 / 官網 / 招募）
2. 價值：你發現了什麼、能幫什麼
3. CTA：明確下一步（回覆 / 預約 15 分鐘 / 看報告）

**系統**：用 AI 生成時，勾選「引用網站問題」，並依產業選對語氣（設定 → AI 知識庫 → 產業模板）。

### 2.2 第三天怎麼追

**觸發**：發信後第 3 天、未回覆  
**系統**：`auto-followup` 預設流程會自動排程。

**話術重點**：
- 不重複第一封內容
- 換角度：例如「上次提到官網，這次附上 3 分鐘摘要」
- 降低門檻：改問「只要回覆『有興趣』即可」

**範本**（系統內建「第 3 天追蹤」）：
> {name} 您好，前幾天曾與您聯繫，想確認是否有機會進一步交流。若您對我們的服務有興趣，歡迎回覆或預約 15 分鐘免費諮詢。

### 2.3 第七天怎麼收尾

**觸發**：第 3 天追蹤後仍無回覆  
**話術**：最後一次、不糾纏、留後門

**範本**（系統內建「第 7 天追蹤」）：
> {name} 您好，這是我們最後一次跟進。若您之後有網站或資安相關需求，歡迎隨時聯繫。祝商祺

**之後**：標記為「冷名單」，6 個月後可再試一次。

### 2.4 已讀不回怎麼改話術

| 情境 | 改法 |
|------|------|
| 開信但未回 | 強調「報告已準備好」「3 分鐘摘要」→ 用 `report-generator.toHtml(report, 'executive')` |
| 點擊報告但未預約 | 發提醒：「您已看過報告，若有疑問可約 15 分鐘聊聊」 |
| 回覆「忙」「之後」 | 3 天後輕量跟進，不推銷，只問「最近還好嗎」 |
| 回覆「誰」「什麼公司」 | 用 `ai-assistant.suggestNextStep()` → 簡短自我介紹 + 價值主張 |

**系統**：開信、點擊由 `/api/track`、`/api/redirect` 記錄，KPI 儀表板可看開信率、點擊率。

## 三、會議 SOP

### 3.1 第一次會議問什麼

**目標**：確認需求、建立信任、判斷是否值得報價。

| 階段 | 問題 | 目的 |
|------|------|------|
| 開場 | 最近在忙什麼？公司/網站/資安現況？ | 破冰、了解現狀 |
| 需求 | 目前最困擾的 3 件事？ | 挖痛點 |
| 預算 | 今年有編這類預算嗎？大概區間？ | 過濾、避免浪費時間 |
| 決策 | 這類專案誰拍板？需要跟誰討論？ | 找對人 |
| 時程 | 希望什麼時候開始？有 deadline 嗎？ | 判斷 urgency |

### 3.2 如何挖痛點

**問法**：
- 「你們現在官網/資安/SEO 最大的問題是什麼？」
- 「如果 3 個月內不處理，最糟會怎樣？」
- 「之前有找過別人做嗎？結果如何？」

**傾聽**：痛點關鍵字 → 對應你的方案（網站重做 / 資安健檢 / SEO / 顧問）。

### 3.3 怎麼帶到你的方案

**結構**：痛點 → 解法 → 案例 → 下一步

1. **痛點**：重複對方說的，確認理解
2. **解法**：我們的做法如何對應（不講技術細節）
3. **案例**：同產業或類似情境的結果（用 AI 知識庫的案例）
4. **下一步**：報價 / 提案 / 再約一次

**避免**：一次講太多功能，聚焦 1–2 個最相關的。

### 3.4 怎麼避免被當免費顧問

| 做法 | 說明 |
|------|------|
| 先報價再深入 | 第一次會議尾聲給價格區間，不給完整解法 |
| 報告當鉤子 | 3 分鐘摘要、一頁提案可給，完整健檢/顧問要收費 |
| 設邊界 | 「這部分屬於顧問範圍，我們有付費方案」 |
| 不寫規格 | 不幫他寫 RFP、不幫他比價，只講我們能做什麼 |

## 四、報價 SOP

### 4.1 哪類需求對應哪個價格帶

| 需求類型 | 價格帶 | 交付物 |
|----------|--------|--------|
| 網站重做（Landing / 形象站） | 5–15 萬 | 響應式網站、基本 SEO |
| 電商 / 複雜功能 | 15–50 萬+ | 依功能報價 |
| 資安健檢（單次） | 2–5 萬 | 報告 + 建議 |
| 資安顧問（年度） | 10–30 萬 | 定期檢查、合規諮詢 |
| SEO / 文案（月費） | 1–3 萬/月 | 關鍵字、內容、監測 |

### 4.2 什麼情況給基礎版

- 預算明確有限（如 &lt; 10 萬）
- 需求單純（只要官網、只要健檢）
- 對方是中小企業、第一次合作

**基礎版**：範圍清楚、交付明確、不包顧問、不包無限修改。

### 4.3 什麼情況給顧問版

- 預算較高（&gt; 20 萬）
- 需求複雜、需長期配合
- 對方有合規、資安、策略需求

**顧問版**：含定期會議、建議、優先支援，價格較高但利潤較好。

### 4.4 什麼情況不接

| 情況 | 理由 |
|------|------|
| 預算差太多 | 報 15 萬他只想花 3 萬 → 不硬做 |
| 決策者不明 | 一直說要問老闆、問股東 → 等決策鏈清晰再談 |
| 只想比價 | 要你寫規格給他去比 → 不當免費顧問 |
| 態度差、不尊重 | 省下時間打更好的客戶 |
| 需求與你專長不符 | 誠實說「這不是我們強項」並轉介 |

## 五、每日檢查清單

| 項目 | 動作 |
|------|------|
| 名單 | 今日新增 10–30 筆，優先 P1 來源 |
| 分數 | 篩選 website_need_score / security_need_score ≥ 60 |
| 發信 | 第一封給新名單，第 3/7 天追蹤給排程到期的 |
| 開信 | 看 KPI 儀表板，已開信未回 → 改話術再追 |
| 會議 | 確認今日會議、準備痛點問題 |
| 報價 | 會議後 24 小時內給報價或提案 |
